What is Machine Learning?

  • Using and building algorithms that learn from data using statistical techniques
  • More information (observations, experiences, instances) should result in higher performance, i.e.the model improves when more information is received
  • Statistical learning techniques allow for identication of patterns within data
  • For the Education Data Strategist, machine learning will focus on making predictions based on the identified patterns

Types of Machine Learning Problems

  • Classification: Predicting the category (or class) of a new observation
    • Qualitative or quantitative output, predefined categories (classes)
  • Clusterting: Group similar objects or observations in clusters, but the clusters are dissimilar from each other
    • Dont need knowledge about the labels, no right or wrong as clustering reveals more information about your data, but there may be an expectataion of a certain result based on prior knowledge
  • Regression: Predicting a continous, quantitative value of a new observation
    • Fitting a function, quantitative output, continuous, trying to find good values for your coefficients, output is based on prior modeled observations

Supervised Learning vs Unsupervised Learning

  • Supervised: Observations are labeled
    • Can compare predicted labels to lables not predicted
  • Unsupervised: Observations are not labeled
    • Clustering is Unsupervised Learning, as labeled observations are not required
    • No labels to compare to
  • Semi-Supervised: Some observations are labeled, most are not.

Lets Build a Machine Learning Model!

Vignette on Binary-Class Classification Problem

  • 87% of Uplift Education graduates enroll in college
  • After graduating, about 10% of our total graduates per graduation year end up leaving college without completing it
  • We wanted to create a model to predict which students were unlikely to enroll for their sophomore year
  • This would allow our Alumni Counselors to spend more time working with these students, so as to insulate them against factors contributing to them dropping out

Steps

  1. Identify Data Set
  2. Determine what type of model we may want to use
  3. Feature Selection
  4. Split Data Set into training data and test data
  5. Train model on training data set
  6. Evaluate model on test data set
  7. Repeat using different model/feature set
  8. Compare models and pick the best one

2. Type of Model to Use

  • Based on what we learned, I would say this is a Classification Problem
  • Therefore, Im going to use a classification model called Classification and Regression Trees (CART)
  • Useful because it can handle both numeric and categorical data implicitly - giving flexibility in our features aka minimal data preparation
  • Also easy to visualize, making it a bit simpler to understand
  • Downside is that prediction performance is often poor due to high variance

    • Two or Four Year College
    • High School the Student Attended

4. Splitting the data

  • Best practice is to split our data into a training set and a test set, so we can build our model on one set of data and evaluate its effectiveness on another set of data
  • In this case, we split the data 70/30 for training and test, giving us 260 observations in our training set and 111 observations in our test set

5. Reading the Decision Tree

  • If the answer to the criteria is yes, follow to the left
  • If the answer to the criteria is no, follow to the right
  • At each decision point, called a node, we can see the distribution of our training set that splits or branches into each yes or no categorization.
  • We follow down until we reach the terminating node, called the leaf
  • If we want to remove certain criteria, we can prune the tree, removing leafs that dont provide enough information

Wait, how did this Decision Tree get built?

  • In this model, the math behind it is called the Gini impurity
  • It determines the highest value splits based on this algorithm
  • The math behind this is not simple, but less complex than other machine learning models - but more on that later

6. How to Evaluate if a Machine Learning Model is Good/Useful

  • Classification
    • Confusion Matrix, Accuracy and Error (Number of times the system is right or wrong), Area Under the Curve (AUC)
  • Clustering
    • Similarity within cluster (want to be high) and similarity between clusters (want to be low)
    • Within cluster sum of squares, diameter, between cluster sum of squares, intercluster distance
  • Regression
    • Root Mean Square Error, R-squared

6. The Reciever Operator Characteristic (ROC) Curve

  • We can also output a probability that an unseen observation is in a particular class via the decision tree or k-nearest neighbor methods.
  • This allows us to define the probability threshold, also known as the Decision Function
  • Confusion Matrix is used to construct the ROC curve, by using the true positive Rate (or Recall) and the false positive rate

6. The ROC Curve Continued

  • Need a classifier that outputs class probabilities - called the Decision Function
  • True Positive Rate goes on the y-axis
  • False Positve Rate goes on the x-axis
  • The Closer the curve is to the upper left corner, the better the classifier is
  • Therefore you want to maximize the Area Under the Curve (AUC). A number closer to 1 is better, and allows you to compare different models

Now What?

  • We just walked through the process of building a classification and regression tree (CART), a type of classification model
  • Imagine doing this process iteratively over and over again, adding more data and resampling that data uing k-fold cross validation - randomly partitioning the data into k-sized subsamples and retraining and testing the model.
  • Ideally, the model performance will continue improve with more data, and allow the model to better generalize to unobserved data
  • We might also want to try other types of models to see if they improve our ability to correctly classify the data

Words of Caution

  • Beware overfitting the data
  • Recognize cases where a feature might have a new type of observation - ex. a graduating class from a new high school
  • Poor sampling of data will lead to a biased model

More Words of Caution

  • Big B Biases are present in our data - particularly in Education Data!
  • Models that predict inequitable outcomes - something seen in the model we just built - are typically due to data that reflect these inequitable outcomes
  • Be careful in choosing your features - there is not a hard and fast mathematical rule you can apply to detect features that are rooted in inequity
  • If a model was used to give out loans from a bank, and race or ethnicity were a feature that decided what interest rate to offer to clients, then the model would be discriminatory
  • Can you think of situations where you might want to proceed with caution when dumping data into a model and just using the output?

More Words of Caution

  • Machine learning models, depending on their complexity, are difficult to explain
  • For this overview, the explanations are simplified to make them accessible
  • There is a black box nature to machine learning, in that some of models that produce the greatest accuracy - like Neural Networks and Stacked Models, are incredibly difficult for data scientists to explain
  • So how are you going to explain a model you struggle to understand to the end user?

So why should we even use Machine Learning?

  • This is another tool in your tool kit to reveal patterns in your data.
  • If the patterns you reveal allow your district to make better decisions, or make predictions more accurate than human intuition, these are very useful.
  • Used in a production setting, then can also be combined with human intuition to guide staff through a decision-making process.
  • It is worth knowing about machine learning, as they algorithms underpin many tools we are using in education - like ed tech and adaptive tests - and our every day lives.
